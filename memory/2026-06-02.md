### 3. Made Hero Stats Strip Separators Symmetric
- **Problem:** The stats strip below the Hero had a vertical border before the first item (`32+`), but not after the last item (`4.9/5`). The Tailwind pseudo-class border layout (`border-s odd:border-s-0 md:odd:border-s md:first:border-s-0`) caused conflicts on medium screens (where the first item is both `first` and `odd`), rendering a left border on the first item and making the stats bar look asymmetrical.
- **Solution:** Replaced the CSS pseudo-class logic in [Hero.tsx](file:///c:/Users/karim/Desktop/islamteban/components/sections/Hero.tsx#L183-L191) with robust, dynamic React index checking:
  ```tsx
  i === 0 ? 'border-s-0' : i % 2 === 0 ? 'border-s-0 md:border-s' : 'border-s'
  ```
  This ensures:
  - The first item (index 0) never has a left border.
  - On mobile (2 columns), only the 2nd and 4th columns get a left border (dividing the columns symmetrically: `col1 | col2`).
  - On desktop (4 columns), columns 2, 3, and 4 get a left border, leaving the edges completely clean (`col1 | col2 | col3 | col4`).
