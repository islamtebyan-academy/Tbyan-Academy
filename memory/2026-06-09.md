## Lessons Learned

- Always bound absolute decorative glow graphics or grids inside containers utilizing `overflow-x-hidden` or enforce it globally at the layout level (`html, body`) in responsive projects.
- Combining `w-full` and `left-0 right-0` on `fixed` position elements can cause width computation issues in some mobile viewports (especially under RTL layout modes), leading to shifted alignments. Enforce widths with edge-anchoring positioning.
- Light text colors like `#6B6052` (stone) or low opacity shades need careful checking under contrast ratio metrics when displayed on light backgrounds (`#FDFAF3` ivory). Using slightly darker hues like `text-navy` or `text-midnight` with higher opacities maintains a premium design aesthetic while ensuring high accessibility.
- Bind new Google fonts to existing CSS variables (`--font-cormorant`, etc.) to update typography site-wide in Next.js instantly and safely, without needing risk-prone, massive manual class refactoring in component files.
